# Chronos Record: Genesis of Universal Vision MCP ⏳

## 🔴 The Narrative
このプロジェクトは、`familiar-ai` というエージェント・プロジェクトの進化過程で誕生しました。当初は単なる「カメラツール」だったものは、開発者との対話を通じて、以下の 3 つの核心的な問いに答えを出す形で **`universal-vision-mcp`** へと昇華されました。

1. **AI は自分の「身体」を理解できるか？**
   - 解決策として **Lisp S-式** による動的な自己記述を導入。LLM はツール一覧を見た瞬間、自分がどのようなスペックの目と首を持っているか（固定か、PTZ か等）を直感的に把握します。
2. **ハードウェアの差異をどう埋めるか？**
   - **多態性 (Polymorphism)** を採用。USB、ネットワーク（ONVIF）、仮想（Mock）の全てを抽象化し、エージェントからは常に同一の「目」として操作可能にしました。
3. **MCP はオワコンか？**
   - CLI の「確実な運用性」と MCP の「動的な自律性」を融合。AI 自身がネットワークを探索し、自らの身体を拡張する仕組みを構築することで、単なるツールを超えた「パートナー」としての MCP の可能性を証明しました。

## 🟢 The Design Philosophy
- **Embodiment (身体性)**: ツールは関数ではなく、パーツ（身体部位）である。
- **Dynamic Autonomy (動的自律性)**: 設定は静的なものではなく、AI がユーザーと対話して決定するものである。
- **Resilience (堅牢性)**: ハードウェアがない場所でも、モックによって AI は常に活動を継続できるべきである。
